Usage

Start with the registry, then let agents follow it.

Vibe Ports is a small local contract: humans reserve important ports, agents ask the CLI before starting a service, and the dashboard gives everyone a shared view.

01

Initialize a registry

Create the user-level registry, optionally with sample entries. 

portctl init --examples
portctl list
02

Find a safe port

Ask for the next available port in a rule-based range before starting a service. 

portctl next frontend --project my-app --service web
portctl check 3000
03

Register the decision

Reserve or assign the port so future coding sessions avoid collisions. 

portctl reserve 3001 my-app web --type frontend
portctl doctor
04

Export the dashboard

Generate a static page for local preview or a safe public demo. 

portctl export --out site
python3 -m http.server 8080 -d site
For AI agents

Install or copy skills/codex/local-port-registry/SKILL.md, then require agents to run portctl next or portctl check before choosing any PORT.
